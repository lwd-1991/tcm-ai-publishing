# 字符编码转换功能说明

## 功能概述

TXT编辑器新增了字符编码转换功能，支持多种编码格式之间的转换，并实时显示转换进度。

## 新增特性

### ✨ 进度百分比显示
- 📊 实时进度条显示
- 🎯 百分比数字显示
- 📝 当前步骤状态提示
- 🎨 渐变色进度条动画

### 🔄 支持的编码格式
- UTF-8（默认）
- GBK
- GB2312
- GB18030
- BIG5（繁体中文）
- Shift_JIS（日文）
- EUC-KR（韩文）

### 🎬 转换过程展示

**进度阶段：**
1. **0-10%** - 准备转换、读取文件内容
2. **10-30%** - 分析编码格式
3. **30-70%** - 执行编码转换
4. **70-90%** - 处理字符映射
5. **90-100%** - 更新编辑器、完成转换

**状态提示：**
- "准备转换..."
- "读取文件内容..."
- "分析编码格式..."
- "从 UTF-8 解码..."
- "转换到 GBK..."
- "处理字符映射..."
- "写入编辑器..."
- "转换完成！"

## 使用方法

### 1. 打开转码对话框
- 点击工具栏的"转码"按钮
- 或使用快捷键（如果配置）

### 2. 选择编码格式
- **当前编码**：文件的原始编码
- **目标编码**：要转换到的编码

### 3. 选择处理方式
- **直接转换当前内容**：转换编辑器中的文本
- **重新打开文件**：使用新编码重新读取文件

### 4. 开始转换
- 点击"开始转换"按钮
- 观察进度条实时更新
- 等待转换完成

### 5. 完成
- 自动关闭对话框
- 显示成功提示
- 编辑器显示转换后的内容

## 重要说明

### ⚠️ 浏览器限制

**当前实现的限制：**
- 浏览器原生 TextEncoder/TextDecoder 主要支持 UTF-8
- 其他编码的完整支持需要外部库
- 当前版本主要用于演示转换流程

### 🔧 完整编码转换方案

如需真正的编码转换，建议使用以下库：

**1. encoding.js**
```html
<script src="https://cdn.jsdelivr.net/npm/encoding-japanese@2.0.0/encoding.min.js"></script>
```

```javascript
// 使用示例
const detected = Encoding.detect(bytes);
const unicodeArray = Encoding.convert(bytes, {
    to: 'UNICODE',
    from: detected
});
```

**2. iconv-lite（Node.js）**
```javascript
const iconv = require('iconv-lite');
const buf = iconv.encode(str, 'gbk');
const str2 = iconv.decode(buf, 'utf8');
```

**3. TextDecoder API（浏览器）**
```javascript
// 支持的编码有限
const decoder = new TextDecoder('gbk');
const text = decoder.decode(buffer);
```

### 📦 推荐集成方案

**方案一：使用 encoding.js**
```html
<!-- 在 head 中引入 -->
<script src="https://cdn.jsdelivr.net/npm/encoding-japanese@2.0.0/encoding.min.js"></script>

<script>
function convertEncoding(text, fromEncoding, toEncoding) {
    // 转换为字节数组
    const bytes = [];
    for (let i = 0; i < text.length; i++) {
        bytes.push(text.charCodeAt(i));
    }
    
    // 检测编码
    const detected = Encoding.detect(bytes);
    
    // 转换编码
    const unicodeArray = Encoding.convert(bytes, {
        to: 'UNICODE',
        from: fromEncoding
    });
    
    // 转换为目标编码
    const converted = Encoding.convert(unicodeArray, {
        to: toEncoding,
        from: 'UNICODE'
    });
    
    // 转换为字符串
    return Encoding.codeToString(converted);
}
</script>
```

**方案二：服务器端转换**
```javascript
// 前端发送请求
async function convertOnServer(content, from, to) {
    const response = await fetch('/api/convert', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ content, from, to })
    });
    return await response.text();
}

// 后端处理（Node.js + iconv-lite）
app.post('/api/convert', (req, res) => {
    const { content, from, to } = req.body;
    const buffer = iconv.encode(content, from);
    const result = iconv.decode(buffer, to);
    res.send(result);
});
```

## 错误处理

### 常见错误

**1. 编码不支持**
```
错误：编码转换失败，可能是内容包含无法转换的字符
解决：检查源编码是否正确，或使用其他编码
```

**2. 字符无法映射**
```
错误：某些字符在目标编码中不存在
解决：使用 UTF-8 作为目标编码，或接受字符丢失
```

**3. 文件过大**
```
错误：转换超时或内存不足
解决：分块处理或使用服务器端转换
```

### 错误提示

- 进度条变为红色
- 显示错误信息
- 保持对话框打开
- 允许重新尝试
