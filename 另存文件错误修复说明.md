# 另存文件错误修复说明

## 问题描述

**错误信息：**
```
保存失败: Failed to execute 'showSaveFilePicker' on 'Window': 
Extension '*' must start with '.'.
```

**原因分析：**
在使用 File System Access API 的 `showSaveFilePicker()` 方法时，文件类型配置中使用了不符合规范的扩展名 `'*'`。根据 Web 标准，所有文件扩展名必须以 `.` 开头。

## 修复方案

### 修复前代码

```javascript
const handle = await window.showSaveFilePicker({
    suggestedName: defaultName,
    types: [
        {
            description: '文本文件',
            accept: { 'text/plain': ['.txt'] }
        },
        {
            description: '所有文件',
            accept: { '*/*': ['*'] }  // ❌ 错误：扩展名必须以 . 开头
        }
    ]
});
```

### 修复后代码

```javascript
const handle = await window.showSaveFilePicker({
    suggestedName: defaultName,
    types: [
        {
            description: '文本文件',
            accept: { 'text/plain': ['.txt'] }
        },
        {
            description: 'Markdown 文件',
            accept: { 'text/markdown': ['.md'] }
        },
        {
            description: 'JSON 文件',
            accept: { 'application/json': ['.json'] }
        },
        {
            description: 'JavaScript 文件',
            accept: { 'text/javascript': ['.js'] }
        },
        {
            description: 'HTML 文件',
            accept: { 'text/html': ['.html', '.htm'] }
        },
        {
            description: 'CSS 文件',
            accept: { 'text/css': ['.css'] }
        },
        {
            description: 'XML 文件',
            accept: { 'text/xml': ['.xml'] }
        },
        {
            description: 'CSV 文件',
            accept: { 'text/csv': ['.csv'] }
        }
    ]
});
```

## File System Access API 规范

### 正确的配置格式

```javascript
{
    description: '文件描述',
    accept: {
        'MIME类型': ['.扩展名1', '.扩展名2']
    }
}
```

### 扩展名规则

**✅ 正确示例：**
```javascript
['.txt']           // 单个扩展名
['.html', '.htm']  // 多个扩展名
['.js', '.jsx']    // JavaScript 相关
```

**❌ 错误示例：**
```javascript
['*']              // 不能使用通配符
['txt']            // 缺少点号
['.']              // 只有点号
['']               // 空字符串
```

### MIME 类型对照表

| 文件类型 | MIME 类型 | 扩展名 |
|---------|----------|--------|
| 文本文件 | text/plain | .txt |
| Markdown | text/markdown | .md |
| JSON | application/json | .json |
| JavaScript | text/javascript | .js |
| HTML | text/html | .html, .htm |
| CSS | text/css | .css |
| XML | text/xml | .xml |
| CSV | text/csv | .csv |
| Python | text/x-python | .py |
| Java | text/x-java | .java |
| C/C++ | text/x-c | .c, .cpp, .h |

## 使用说明

### 另存文件流程

1. **打开编辑器**
   - 编辑或打开文件

2. **点击另存为**
   - 点击工具栏"另存为"按钮

3. **选择文件类型**
   - 在保存对话框中选择文件类型
   - 可选：文本文件、Markdown、JSON 等

4. **输入文件名**
   - 输入文件名（自动添加扩展名）
   - 或保留默认文件名

5. **选择保存位置**
   - 选择保存目录
   - 点击"保存"按钮

6. **保存成功**
   - 显示成功提示
   - 文件状态更新

## 浏览器兼容性

### File System Access API 支持

**✅ 支持的浏览器：**
- Chrome 86+
- Edge 86+
- Opera 72+

**❌ 不支持的浏览器：**
- Firefox（使用下载方式）
- Safari（使用下载方式）
- IE（不支持）

### 降级方案

对于不支持 File System Access API 的浏览器，编辑器会自动降级到下载方式：

```javascript
if ('showSaveFilePicker' in window) {
    // 使用 File System Access API
    // 可以直接保存到指定位置
} else {
    // 降级到下载方式
    // 文件会下载到默认下载目录
    downloadFile(content, filename);
}
```

## 错误处理

### 常见错误及解决方案

**1. AbortError**
```
原因：用户取消了保存操作
处理：不显示错误提示
```

**2. NotAllowedError**
```
原因：用户拒绝了文件访问权限
处理：提示用户授予权限
```

**3. SecurityError**
```
原因：安全策略限制
处理：降级到下载方式
```

**4. TypeError**
```
原因：配置参数错误
处理：检查文件类型配置
```

### 错误处理代码

```javascript
try {
    const handle = await window.showSaveFilePicker({...});
    // 保存文件
} catch (error) {
    if (error.name === 'AbortError') {
        // 用户取消，不显示错误
        return;
    }
    if (error.name === 'NotAllowedError') {
        showToast('请授予文件访问权限', 'error');
        return;
    }
    // 其他错误
    showToast('保存失败: ' + error.message, 'error');
}
```
